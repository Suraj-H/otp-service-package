# First commit on `main` (bootstrap checklist)

Use this for a **clean initial import** when the repo has no commits yet. It matches [phase-pr-playbook.md](phase-pr-playbook.md) (“already-unified tree” case): **one** honest commit, then phase-style PRs afterward.

## 1. Verify locally

```bash
pnpm install
pnpm lint && pnpm typecheck && pnpm test && pnpm verify:publishability
```

## 2. Do **not** commit (unless you explicitly want them public)

| Path | Why |
|------|-----|
| `agent-skills/` | Ignored in [.gitignore](../../.gitignore) — local skill/plugin copy, not the library |
| `node_modules/`, `dist/`, `coverage/`, `.pnpm-store/` | Generated / install output (see [.gitignore](../../.gitignore)) |
| `.env` | Secrets |
| `tests/publishability/tarballs/` | Regenerated by scripts (ignored) |
| `tests/publishability/consumer/node_modules` | Fixture install output (ignored) |

## 3. **Do** commit (monorepo product)

Everything that defines the library, CI, docs, and examples **except** the exclusions above.

**Copy-paste staging (paths only — no secrets):**

```bash
git add \
  .changeset/ \
  .env.example \
  .github/ \
  .gitignore \
  .husky/pre-commit \
  AGENTS.md \
  CONTRIBUTING.md \
  LICENSE \
  README.md \
  docs/ \
  eslint.config.mjs \
  examples/ \
  package.json \
  packages/ \
  pnpm-lock.yaml \
  pnpm-workspace.yaml \
  scripts/ \
  tests/ \
  tooling/ \
  tsconfig.base.json \
  tsconfig.eslint.json \
  tsconfig.json \
  tsconfig.package.json \
  tsconfig.test.json \
  vitest.config.ts
```

Then review:

```bash
git status
git diff --staged --stat
# Optional: scan for accidental secrets
git diff --staged | grep -iE 'password|secret|api_key|token|BEGIN (RSA |OPENSSH )?PRIVATE' || true
```

## 4. Commit message (Conventional Commits)

```text
chore: initial monorepo import

OTP package family: core, redis-store, testkit, Twilio/Resend adapters,
Express/Fastify/Nest, starter, examples, integration tests, CI, Changesets,
and docs. See docs/plans/otp-library-implementation-plan.md.
```

## 5. Push and open first PR (if using PR flow)

```bash
git commit -m "chore: initial monorepo import"
git remote add origin <your-github-url>   # if missing
git push -u origin main
# or: gh repo create ... && git push -u origin main
```

## 6. After `main` exists

- Run **`pnpm sync-repo-url`** so `repository` / `bugs` / `homepage` match `origin`.
- Add GitHub secret **`NPM_TOKEN`** before relying on [release workflow](../../.github/workflows/release.yml).
- Prefer **branch protection** requiring the CI check on `main`.

Future work: follow [phase-pr-playbook.md](phase-pr-playbook.md) for phase-sized branches and PRs.
